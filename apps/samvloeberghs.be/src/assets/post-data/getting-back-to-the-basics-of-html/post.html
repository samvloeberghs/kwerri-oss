For the past 3 years ( and probably some more years to come ) I've been in roles where I take the role as technical lead. For me this role is not about telling people what to do, but trying to lead the team in using the correct technologies and applying them keeping in mind best practices.

The last projects I worked on all used Angular v2+. The framework allows us to implement business functionality at a rapid pace but I noticed that many developers are forgetting about the basics that surround Angular. And one of those basics is just plain old simple HTML. But is it that simple? Using it badly is, unfortunately..

A pretty nasty disease called Divitis

Many developers, mostly the ones fairly new to webdevelopment, diving immediatly into a big framework like Angular or React quickly suffer from a disease called Divitis. Divitis is a rather old term and is referenced by Wikipedia as:

The practice of authoring web-page code with many div elements in place of meaningful semantic HTML elements.

It seems that the first occurence of Divitis dates back to 2003 in a book by Jeffrey-Zeldman, called "Designing with Web Standards". This doesn't make it less actual. The core & base of HTML elements has not been changed dramatically the last 2 decades, except for the introduction of more semantic elements with HTML5 like <header> or <footer> and others. More information and a nice overview can be found in this nice article by Matt West. The disease of Divitis can only be cured by the medicine of learning :)

Some common mistakes & bad-practices

While doing code reviews of HTML of the projects I work on I come accross mistakes & bad-practices that come back every time. Writing this blogpost also served as base to get my own HTML structure in order :)

Correct emphasizing

Sometimes the <b> & <i> tags are used for emphasizing text. Although it's possible to change the styles of these elements using CSS, doing it would not make sense. These tags are typically refered to as presentational tags. The <b> tag refers to bold and the <i> tag refers to italic. Changing the CSS would change the semantic meaning of the tag.

Using the <strong> & <em> tags defines more semantic meaning to what your emphasizing. The <strong> tag is mainly used to stress the importance of an entire sentence, while <em> is typically used to stress the importance of one word in a longer text or sentence.
